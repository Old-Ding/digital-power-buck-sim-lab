# 【数字电源仿真入门 01】为什么从 Buck 开始做 MATLAB + PLECS 仿真

## 前言

我准备写一个数字电源仿真入门系列。

这个系列不打算一上来就讲很复杂的拓扑，也不打算把 MATLAB、PLECS 当成孤立的软件教程来写。我的目标是用一个完整的小项目，把开关电源软件工程师需要掌握的几件事串起来：

```text
功率级建模
数字控制
PWM 占空比
ADC 采样
软启动
过压/过流/欠压保护
状态机
仿真波形分析
C 风格控制代码
```

项目名称暂定为：

```text
digital-power-buck-sim-lab
```

第一阶段先做一个最典型、最容易跑通主线的项目：

```text
24V 输入
12V / 5A 输出
Buck 降压电源
数字 PI 电压环控制
```

这篇是系列第 1 篇，先不急着画电路，也不急着调 PI 参数。本文先把学习路线、项目边界和整体框架讲清楚。

## 一、这个系列适合谁

这个系列主要适合几类人：

```text
1. 想入门开关电源软件开发的人
2. 学过一点 Buck/Boost，但不知道怎么做闭环控制的人
3. 会一点 STM32 或 C 语言，但不知道电源控制软件怎么组织的人
4. 有 MATLAB/PLECS，但不知道怎么做一个完整项目的人
5. 想做一个能放到 GitHub 和简历里的数字电源作品集的人
```

如果你已经能熟练设计 PFC、LLC、移相全桥，或者已经在公司做数字电源量产项目，那这个系列会比较基础。

但如果你的状态是：

```text
知道 PWM
知道 ADC
知道 Buck
知道 PI
但是不知道它们怎么组成一个能工作的电源控制系统
```

那这个系列就是从这个断点开始补。

## 二、为什么不直接学复杂电源拓扑

很多人一提开关电源，就想直接学：

```text
反激电源
LLC
PFC
移相全桥
三相逆变
```

这些当然重要，但不适合作为第一个数字电源软件项目。

原因有三个。

第一，复杂拓扑会引入太多额外问题。

比如反激会涉及变压器、隔离反馈、漏感、吸收电路；LLC 会涉及谐振腔、频率控制、增益曲线；PFC 会涉及输入电流整形、母线电压控制、工频周期采样。

这些内容都很重要，但它们会把初学阶段的主线打散。

第二，第一阶段最需要建立的是控制链路，而不是挑战拓扑复杂度。

数字电源软件最基础的链路是：

```text
采样输出电压
-> 和目标电压比较
-> 控制器计算
-> 更新 PWM
-> 功率级响应
-> 再次采样
```

这个链路在 Buck 里最容易看清楚。

第三，低压 Buck 更安全。

第一版项目只做低压 DC-DC，不碰 220V 市电。这样可以把精力放在建模、控制和测试上，而不是安全风险和硬件保护上。

## 三、为什么第一版选 Buck

第一版项目规格如下：

```text
输入电压：18V 到 30V
标称输入：24V
输出电压：12V
最大输出电流：5A
最大输出功率：60W
拓扑：Buck
控制方式：数字 PI 电压环
初始开关频率：200kHz
```

Buck 的好处是简单直接。

理想情况下，Buck 的输出电压和占空比大致满足：

```text
Vout ≈ D * Vin
```

其中：

```text
Vout：输出电压
D：占空比
Vin：输入电压
```

举个例子，输入是 24V，希望输出 12V，那么理想占空比大约是：

```text
D = Vout / Vin = 12 / 24 = 0.5
```

当然，真实电路里会有开关损耗、电感电阻、电容 ESR、二极管压降或同步 MOSFET 导通损耗，所以不会完全等于 0.5。

但这个关系足够清晰，很适合用来建立第一版模型。

第一阶段我们要先验证三件事：

```text
1. 固定占空比时，输出电压趋势是否正确
2. 电感电流是否符合 Buck 工作规律
3. 改变负载和输入电压时，模型响应是否合理
```

只有功率级本身是可信的，后面加 PI 控制才有意义。

## 四、MATLAB 和 PLECS 怎么分工

这个项目会同时用 MATLAB/Simulink 和 PLECS。

它们不是互相替代的关系，而是分工不同。

### 1. PLECS 负责功率级

PLECS 更适合做电力电子电路仿真，比如：

```text
MOSFET
二极管
电感
电容
ESR
负载变化
输入电压扰动
开关节点波形
```

所以 Buck 主电路优先放在 PLECS 里。

第一版 PLECS 模型至少要看到这些波形：

```text
输入电压 Vin
输出电压 Vout
电感电流 IL
开关节点 SW
占空比 duty
输出电流 Iout
```

### 2. MATLAB/Simulink 负责控制和测试

MATLAB/Simulink 更适合做：

```text
离散 PI 控制
软启动参考值
参数扫描
测试矩阵
波形处理
报告生成
```

后面会用脚本统一跑测试，比如：

```text
空载启动
满载启动
20% 到 100% 负载突变
输入电压从 24V 降到 18V
输出过流故障
输出过压故障
```

### 3. C 风格代码负责贴近真实固件

只做仿真还不够。

开关电源软件最终还是要落到 MCU 或 DSP 上，比如 STM32G4、TI C2000 这类平台。

所以项目里会整理一套 C 风格控制代码：

```text
pi_controller.c
protection.c
power_state_machine.c
power_control.c
config.h
```

这样做的目的不是马上烧录到板子，而是让控制逻辑从一开始就贴近真实固件结构。

## 六、数字电源软件到底在控制什么

以 Buck 为例，数字电源软件的核心任务是让输出电压稳定在目标值。

可以先把它理解成这样：

```text
目标输出：12V
实际输出：Vout
误差：error = 12V - Vout
控制器根据 error 调整 duty
duty 改变后，Buck 输出随之变化
```

也就是：

```text
Vout 低了 -> duty 增大
Vout 高了 -> duty 减小
```

但真实软件不能只写这两句话。

它还要考虑：

```text
启动时不能一下子给 12V 目标
占空比不能无限增大
积分项不能一直累加
输入欠压时要关断
输出过压时要保护
输出过流时要锁故障
故障发生后要记录原因
调试时要能看到关键变量
```

所以第一版软件会按下面的结构组织：

```text
PI 控制器：
负责电压误差和 duty 计算

保护检测：
负责 OVP、OCP、UVLO、OTP

状态机：
负责 INIT、IDLE、SOFT_START、RUN、FAULT_LATCH、RECOVERY

Power Control：
负责把采样值、状态、故障码、duty 汇总起来
```

这里有一个重要原则：不同职责不要混在一起。

比如过流保护不应该到处判断一遍。第一版只在保护层判断过流，状态机只消费保护层给出的故障码。

这样后面调试时，才能知道谁负责检测、谁负责关断、谁负责锁存。
