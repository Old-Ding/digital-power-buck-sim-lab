# 24V 到 12V/5A 数字 Buck 仿真项目设计

这篇文章是数字电源仿真项目的开篇。项目目标不是简单跑一个 Buck 波形，而是按开关电源软件工程师的视角，把功率级、数字控制、保护状态机、测试矩阵和 C 风格控制代码串成一条完整主线。

项目仓库名暂定为：

```text
digital-power-buck-sim-lab
```

## 为什么选这个项目

如果目标是开关电源软件工程师，只学 MCU 外设或者只看电源拓扑都不够。真实工作里经常要同时面对这些问题：

- PWM 怎么更新占空比；
- ADC 采样怎么进入控制环；
- PI 参数怎么影响动态响应；
- 软启动怎么避免过冲和大电流；
- 过压、过流、欠压、过温怎么处理；
- 故障码和关键变量怎么保留，方便调试；
- 仿真控制器后续怎么迁移到 MCU。

所以第一阶段选一个低压 Buck 项目。Buck 足够简单，能把主线跑通；同时它又足够典型，能覆盖数字电源控制里的核心问题。

## 项目规格

第一版规格如下：

```text
输入电压：18V 到 30V，标称 24V
输出电压：12V
最大输出电流：5A
最大输出功率：60W
拓扑：Buck
控制方式：离散 PI 电压环
初始开关频率：200kHz
保护：UVLO / OVP / OCP / OTP
```

第一阶段不碰 220V AC 输入、不做 PFC、不做 LLC、不做反激。这样做不是因为那些不重要，而是因为第一版项目的目标是把数字电源软件链路做扎实，避免安全风险和目标发散。

## 工具分工

我会使用 MATLAB/Simulink 和 PLECS。它们的职责分工是：

```text
PLECS：负责 Buck 功率级建模、开关波形、输入/负载扰动
MATLAB/Simulink：负责数字控制、测试矩阵、波形处理、报告脚本
C 风格代码：负责把控制逻辑整理成接近 MCU 的工程结构
GitHub：负责项目版本管理和作品集展示
CSDN：负责按问题复盘，记录设计和调试过程
```

## 软件架构

控制逻辑会按职责分层：

```text
PI 控制器：只负责误差计算、占空比限幅、抗积分饱和
Protection：只负责故障检测和故障优先级
State Machine：只负责状态切换和故障锁存
Power Control：汇总采样、状态、控制输出和关键变量
```

这么设计是为了避免把所有逻辑都塞进一个控制器里。保护阈值、状态切换、PI 内部状态属于不同职责，混在一起后期很难调试。

## 第一阶段目标

第一阶段只追求一个清晰闭环：

```text
PLECS Buck 功率级
-> 采样 Vout / Iout / Vin / Temp
-> 离散 PI 电压环
-> 软启动参考值
-> 保护检测
-> 故障状态机
-> 输出 duty
-> 回到功率级
```

每一步都要能观测关键变量：

```text
vin
vout
iout
temperature
vref
duty
state
fault_code
pi_integrator
```

## 测试矩阵

项目不会只看一个稳态波形，后续会覆盖这些工况：

```text
空载启动
满载启动
20% 到 100% 负载突变
100% 到 20% 负载突变
输入电压 24V 降到 18V
输入欠压触发 UVLO
输出过流触发 OCP
输出过压触发 OVP
ADC 噪声导致 duty 抖动
```

每个问题都会按同一个格式记录：

```text
问题是什么
怎么复现
关键变量是什么
波形现象是什么
根因是什么
最小修改是什么
怎么验证
对应 GitHub commit
```
