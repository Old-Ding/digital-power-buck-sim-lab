# 我准备用 MATLAB + PLECS 做一个数字 Buck 电源仿真项目

这不是一篇 Buck 原理入门，也不是 MATLAB/PLECS 工具教程。

我想做的是一个面向开关电源软件工程师的作品集项目：用 PLECS 建功率级，用 MATLAB/Simulink 做数字控制和测试，再把核心控制逻辑整理成接近 MCU 固件的 C 风格代码。

项目仓库名暂定：

```text
digital-power-buck-sim-lab
```

## 为什么做这个项目

开关电源软件工程师这个方向，不能只会嵌入式外设，也不能只会电源拓扑。

只会 MCU，容易停留在：

```text
会配 PWM
会读 ADC
会写中断
```

但不知道这些外设怎么服务于一个闭环电源。

只会电源理论，也容易停留在：

```text
知道 Buck/Boost
知道电感电容怎么估算
知道 PI 控制大概是什么
```

但说不清这些东西怎么变成可运行、可调试、可保护的电源软件。

真实项目里，软件要处理的是一条完整链路：

```text
ADC 采样
-> 数字滤波/标定
-> 控制环计算
-> PWM 占空比更新
-> 软启动
-> 过压/过流/欠压/过温保护
-> 故障锁存和恢复
-> 参数管理
-> 关键变量观测
```

所以我不打算从“学一个工具”开始，而是从一个完整但可控的数字电源项目开始。

## 为什么第一版选 Buck

第一版项目规格定为：

```text
输入电压：18V 到 30V，标称 24V
输出电压：12V
最大输出电流：5A
最大输出功率：60W
拓扑：Buck
初始开关频率：200kHz
控制方式：离散 PI 电压环
保护：UVLO / OVP / OCP / OTP
```

第一版不做 PFC、LLC、反激，也不碰 220V 市电。

这个边界是有意收窄的。Buck 的功率级足够简单，适合先把数字控制、保护状态机和测试方法跑通；同时 Buck 又足够典型，能覆盖开关电源软件最核心的动作：采样、闭环、限幅、软启动、保护和调试。

如果一开始就做复杂拓扑，很容易把时间耗在磁性器件、市电安全、隔离反馈、驱动细节上，反而主线不清楚。

## 工具怎么分工

这个项目里，工具不是目的，工具只承担职责。

```text
PLECS：
负责 Buck 功率级建模，包括开关器件、电感、电容、ESR、负载变化、输入扰动和关键电气波形。

MATLAB/Simulink：
负责数字控制逻辑、参数扫描、测试矩阵、波形分析和报告脚本。

C 风格代码：
负责把控制逻辑整理成接近 MCU 固件的结构，方便后续迁移到 STM32G4 或 TI C2000。

GitHub：
负责版本管理、文档沉淀和作品集展示。

CSDN：
负责按问题复盘，不写流水账。
```

我希望最后看到的不是一个孤立的仿真文件，而是一套可以解释的数据链路、控制结构和测试证据。

## 软件结构怎么设计

第一版控制软件按职责分层：

```text
PI 控制器：
只负责误差计算、占空比限幅、抗积分饱和。

Protection：
只负责故障检测和故障优先级，比如 OCP、OVP、UVLO、OTP。

State Machine：
只负责状态切换和故障锁存。

Power Control：
负责把采样值、状态机、控制器输出和关键变量汇总起来。
```

状态机初版设计为：

```text
INIT
IDLE
SOFT_START
RUN
FAULT_LATCH
RECOVERY
```

这么分层的原因很直接：不同逻辑的职责不同。

PI 控制器关心的是输出电压误差和占空比；保护层关心的是电压、电流、温度有没有越界；状态机关心的是当前电源应该启动、运行、关断还是锁故障。

如果这些判断全部塞进一个函数里，前期看起来省事，后期调试会很痛苦。尤其是故障保护和控制环路互相影响时，很难判断到底是谁改了 duty、谁锁了故障、谁清了积分项。

## 我会重点观察哪些变量

数字电源软件不能只看输出电压。第一版会把这些变量作为核心观测量：

```text
vin
vout
iout
temperature
vref
duty
state
fault_code
pi_integrator
```

这些变量对应不同问题：

```text
vout 看输出是否稳定
iout 看负载和过流
duty 看控制器是否打满
vref 看软启动是否按预期爬升
state 看状态机是否走错
fault_code 看保护是否误触发
pi_integrator 看积分是否饱和
```

后续每次调试都不会只说“波形不对”，而是尽量回到这些变量上找根因。

## 测试矩阵怎么定

第一版不会只跑一个稳态工况。计划覆盖这些测试：

```text
空载启动
满载启动
20% -> 100% 负载突变
100% -> 20% 负载突变
Vin 24V -> 18V
Vin 24V -> 30V
输入欠压触发 UVLO
输出过流触发 OCP
输出过压触发 OVP
ADC 噪声导致 duty 抖动
```

每个测试都按同一个格式记录：

```text
测试目的
输入条件
期望结果
关键波形
异常现象
根因分析
最小修改
验证结果
对应 commit
```

这也是我想通过这个项目训练的核心能力：不是把模型跑起来就结束，而是能解释问题、复现问题、定位问题，然后用最小修改验证假设。

## 阶段目标

第一阶段先做开环 Buck：

```text
固定 duty
-> 观察 Vout 是否接近 Vin * duty
-> 观察电感电流是否符合连续/断续模式预期
-> 观察 SW 节点和输出纹波
```

开环验证通过后，再加离散 PI 电压环：

```text
ADC 采样
-> error = vref - vout
-> PI 计算
-> duty 限幅
-> 抗积分饱和
-> PWM 更新
```

再往后加入软启动和保护状态机：

```text
软启动参考值 0V -> 12V
UVLO / OVP / OCP / OTP 检测
故障锁存
故障清除
关键变量记录
```
